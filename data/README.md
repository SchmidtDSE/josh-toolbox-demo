# Input Data

This directory contains **raw input rasters** for the vegetation simulation.

## Canonical Files

| File | Description | Generated By | Units |
|------|-------------|--------------|-------|
| `fire_severity.tif` | Synthetic fire severity pattern | `demo_workflow.ipynb` cell "generate-fire" | 0-1 ratio |
| `elevation.tif` | Synthetic elevation gradient | `demo_workflow.ipynb` cell "dexdjijy4qv" | 0-100 percent |

## Data Flow

```
data/*.tif  →  preprocessed/*.jshd  →  Josh simulation  →  results/*.csv
   (raw)         (Josh format)           (engine)           (output)
```

## Regenerating Files

Run the following cells in `demo_workflow.ipynb`:
1. **Fire severity**: Run cell "generate-fire" to create `fire_severity.tif`
2. **Elevation**: Run cell "dexdjijy4qv" to create `elevation.tif`

After regenerating TIF files, run the preprocessing cell ("preprocess-fire") to update the `.jshd` files.

## For Production Use

In a real application, these would come from external sources:
- **Fire severity**: Derived from Sentinel-2 RBR (Relativized Burn Ratio) or similar
- **Elevation**: From DEM (Digital Elevation Model) such as SRTM, ASTER, or USGS NED

## Notes

- All rasters use WGS84 (EPSG:4326) coordinate reference system
- Grid dimensions must match `vegetation_model.josh` configuration
- NoData value: -9999
